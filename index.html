المشكلة والحل

المشكلة

التبرعات لا تسجل في القائمة ولا تظهر للمستخدمين الآخرين عند التبرع من أجهزة مختلفة.

هذه المشكلة عادةً ما تكون بسبب:

  • إعدادات أمان Firebase
  • قواعد Firestore غير صحيحة
  • مشكلة في تهيئة Firebase في الصفحة

الحل

لحل هذه المشكلة، نحتاج إلى:

  • تعديل قواعد Firestore للسماح بالقراءة والكتابة
  • التحقق من تهيئة Firebase بشكل صحيح
  • اختبار الاتصال مع Firebase

خطوات الإصلاح

1

تعديل قواعد Firestore

اذهب إلى console.firebase.google.com → مشروعك → Firestore Database → قواعد

استبدل القواعد الحالية بما يلي:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}

ملاحظة: هذه القواعد للاختبار فقط ويجب تقييدها أكثر لأغراض الأمان لاحقًا.

2

3

اختبار الاتصال

لاختبار ما إذا كان الاتصال ب Firebase يعمل correctly، يمكنك استخدام هذا الكود:

// اختبار الاتصال ب Firebase
firebase.initializeApp(firebaseConfig);
const db = firebase.firestore();

// محاولة كتابة بيانات اختبار
db.collection("test").add({
  test: "اتصال ناجح",
  timestamp: firebase.firestore.FieldValue.serverTimestamp()
})
.then(() => {
  console.log("تم الاتصال بنجاح!");
  alert("الاتصال ب Firebase يعمل correctly");
})
.catch((error) => {
  console.error("خطأ في الاتصال: ", error);
  alert("هناك خطأ في الاتصال: " + error.message);
});

إعداد Firebase بشكل صحيح

لضمان عمل Firebase بشكل صحيح، اتبع هذه الخطوات:

1

إنشاء مشروع Firebase جديد

اذهب إلى console.firebase.google.com وأنشئ مشروعًا جديدًا.

2

إضافة تطبيق ويب

من صفحة المشروع، انقر على رمز الويب (+) وأدخل اسم التطبيق، ثم انسخ التكوين.

3

تفعيل Firestore

من القائمة الجانبية، اختر "Firestore Database" وأنشئ قاعدة بيانات جديدة.

4

تعديل القواعد

كما هو موضح سابقًا، غير قواعد Firestore للسماح بالقراءة والكتابة.

اختبار النظام

بعد تطبيق الإصلاحات، يمكنك اختبار النظام من خلال:

  1. فتح الموقع على جهازين مختلفين أو نافذتين مختلفتين
  2. إضافة تبرع اختبار من أحد الأجهزة
  3. التحقق من ظهور التبرع على الجهاز الآخر خلال ثوانٍ

إذا استمرت المشكلة، تأكد من:

  • أنك استبدلت تكوين Firebase بتكوين مشروعك الخاص
  • أن قواعد Firestore تسمح بالقراءة والكتابة
  • أن هناك اتصالاً بالإنترنت على كلا الجهازين

احتياطات أمنية

بعد التأكد من عمل النظام correctly، يجب تعديل قواعد Firestore لتكون أكثر أمانًا:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /donations/{document} {
      allow read: if true;
      allow write: if request.auth != null; // أو شروط أخرى مناسبة
    }
  }
}
